# 기여 가이드

## 시작하기

1. 중복 이슈와 진행 중인 PR을 확인합니다.
2. 최신 `main`에서 규칙에 맞는 개인 브랜치를 생성합니다.
3. 작업 범위를 한 가지로 제한하고 코드, 테스트와 문서를 함께 갱신합니다.
4. 로컬 검증을 실행한 뒤 PR 템플릿을 빠짐없이 작성합니다.
5. CI 통과와 최종 diff 확인 후 squash merge합니다.

현재 solo 모드는 승인 수 0이며 팀 모드는 프로젝트 소유자가 명시적으로 전환합니다.
어느 모드에서도 `main` 직접 push는 허용하지 않습니다.

## PR에 포함할 내용

- 무엇을 바꾸었는지 1~3줄 요약
- 왜 필요한지와 관련 이슈 또는 설계 문서
- 실행한 정확한 검증 명령과 결과
- UI 변경이라면 전후 스크린샷
- 위험, 롤백 방법과 데이터·라이선스 영향

## 데이터 기여

데이터를 추가하거나 재배포할 때 파일별 출처, 권리자, 이용 조건, 재배포 가능 여부,
개인정보·민감정보 여부와 가공 내역을 기록합니다. 공개 접근이 가능하다는 사실만으로
재배포 권리가 생기지는 않습니다.

## 보안

토큰, 비밀번호, 개인정보 또는 실제 취약점 재현 정보를 공개 Issue와 PR에 올리지
마세요. 보안 문제는 [SECURITY.md](SECURITY.md)의 비공개 신고 절차를 사용합니다.
